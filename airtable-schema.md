# Airtable Base Schema

## Base Name: Real Estate Deal Engine

## Table: Real Estate Deals

### Fields Configuration

| Field Name | Field Type | Options/Format | Description |
|------------|------------|----------------|-------------|
| Address | Single line text | - | Full property address |
| Price | Number | Format: Currency ($) | Listing price in USD |
| Property Type | Single select | Options: house, condo, apartment, townhouse, land, commercial, other | Type of property |
| Square Feet | Number | Format: Number | Living area in square feet |
| Bedrooms | Number | Format: Number (decimal) | Number of bedrooms |
| Bathrooms | Number | Format: Number (decimal) | Number of bathrooms (can be decimal like 2.5) |
| Year Built | Number | Format: Number | Year property was built |
| Lot Size | Number | Format: Number | Lot size in square feet |
| City | Single line text | - | City name |
| State | Single line text | - | State (2-letter abbreviation preferred) |
| Zip Code | Single line text | - | ZIP/postal code |
| Description | Long text | - | Brief property description (max 200 chars) |
| Price per Sqft | Number | Format: Number (decimal, 2 places) | Calculated price per square foot |
| Investment Score | Number | Format: Number (integer) | Score from 0-100 based on investment rules |
| Decision | Single select | Options: PASS, REJECT | Final decision (PASS = meets criteria) |
| Decision Reason | Long text | - | Explanation of the decision |
| Processed At | Date & time | Format: ISO 8601 | Timestamp when record was processed |
| Record ID | Formula/Auto | - | Auto-generated by Airtable |

### Recommended Views

1. **All Deals** (Grid view)
   - Default view showing all records
   - Sort by Processed At (newest first)

2. **Passed Deals** (Grid view)
   - Filter: Decision = PASS
   - Sort by Investment Score (highest first)
   - Useful for dashboard display

3. **By Property Type** (Grouped view)
   - Group by Property Type
   - Sort by Investment Score

4. **Recent Deals** (Grid view)
   - Filter: Processed At (last 30 days)
   - Sort by Processed At (newest first)

### Indexes/Performance
- Consider adding a formula field for quick filtering
- Index on "Processed At" for time-based queries
- Index on "Decision" for filtering
